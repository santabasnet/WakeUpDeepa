    How this works
    01 Continuous STT — Web Speech API (`SpeechRecognition`) streams your voice to the browser's built-in speech engine with continuous: true and interimResults: true, so it never stops listening.
    02 Multi-alternative results — up to 3 recognition hypotheses per utterance are checked. Each is scored independently so accent variation is captured across alternatives.
    03 Fuzzy keyword matching — transcripts are matched using Levenshtein edit-distance against every variant: dipa, deepa, deepaa, deepha, namashte, namasthe, hay, hei, hi… Token-level bigram scanning catches split-word results.
    04 Threshold + cooldown — scores above 0.70 fire a detection. A 2.2 s cooldown prevents double-fires. Works in Chrome, Edge, and Safari. Use Chrome for best Nepali phoneme coverage.
